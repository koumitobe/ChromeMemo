---

## 1. 背景・目的

社内メンバーがWebブラウジング中に思いついたメモ・タスク・URLを素早く記録・管理できるChrome拡張機能を提供する。
ストアへの公開は不要で、社内でローカルファイルとして配布・利用する。

---

## 2. スコープ

### In Scope（対象）

- サイドバーUIによる複数メモの管理
- ブラウザローカルへのデータ永続化
- Google Keepへのエクスポート
- キーボードショートカット（Mac / Windows両対応）
- タグ付けと絞り込み
- メモ全文検索
- ピン留め機能
- 作成日時の表示
- 現在のページURLの自動挿入
- JSONエクスポート / インポート
- Markdownプレビュー
- 社内配布用の導入・運用マニュアル

### Out of Scope（対象外）

- Chrome Web Storeへの公開
- メモのクラウド同期・チーム共有
- モバイルブラウザへの対応
- Firefox / Safari等への対応
- Google Keep連携（Phase 2以降の課題）

### 将来課題（Phase 2 検討）

- **Google Keep連携**: OAuth 2.0 外部アプリ・テストモード（社内100名以下）で実装予定。Google Cloud ConsoleでのOAuthクライアントID発行・テストユーザー登録が必要。

---

## 3. ユーザーストーリー

```
As a 社内ユーザー
I want to どのWebページを閲覧中でもサイドバーからメモを管理したい
So that ブラウジングを中断せずに情報を記録・参照できる
```

```
As a 社内ユーザー
I want to メモをGoogle Keepにエクスポートしたい
So that 既存のKeepワークフローとシームレスに連携できる
```

---

## 4. 機能要件

### FR-001: サイドバー表示・非表示

- **説明**: 任意のWebページ上にサイドバーを表示/非表示できる
- **優先度**: Must
- **受け入れ条件**:
  - ツールバーアイコンのクリックでサイドバーが開閉する
  - キーボードショートカットで開閉できる（Mac: `Cmd+M` / Windows: `Alt+M`）
  - サイドバーはページコンテンツに重ならず右端に固定表示される
  - サイドバーの幅はドラッグでリサイズできる
  - ページ移動後も開閉状態が維持される

### FR-002: 複数メモの作成・編集・削除

- **説明**: 複数のメモをリスト形式で管理する
- **優先度**: Must
- **受け入れ条件**:
  - 「新規メモ」ボタンでメモを作成できる
  - メモにはタイトルと本文を入力できる
  - メモを編集・保存できる
  - メモを削除できる（確認ダイアログあり）
  - メモ一覧はリスト形式で表示され、最終更新順に並ぶ

### FR-003: データ永続化

- **説明**: メモデータをブラウザローカルに保存し、ページ移動・ブラウザ再起動後も保持する
- **優先度**: Must
- **受け入れ条件**:
  - `chrome.storage.local` にメモデータを保存する
  - ページ遷移後にサイドバーを開いても最新のメモが表示される
  - ブラウザを再起動してもデータが消えない

### FR-004: キーボードショートカット

- **説明**: Mac / Windows両対応のショートカットでサイドバーを開閉する
- **優先度**: Must
- **受け入れ条件**:
  - Mac: `Command + M` でサイドバーが開閉する
  - Windows / Linux: `Alt + M` でサイドバーが開閉する
  - ショートカットはChrome拡張のコマンド設定（`commands`）で実装する

### FR-005: タグ付けと絞り込み

- **説明**: メモに任意のタグを付与し、タグで絞り込み表示できる
- **優先度**: Should
- **受け入れ条件**:
  - メモ編集時にタグを複数設定できる（例: `#仕事` `#確認`）
  - サイドバー上部でタグを選択するとそのタグのメモのみ表示される
  - タグ一覧は使用中のタグを自動収集して表示する

### FR-006: メモ全文検索

- **説明**: キーワードで全メモのタイトル・本文を横断検索する
- **優先度**: Should
- **受け入れ条件**:
  - 検索ボックスにキーワードを入力すると即時フィルタリングされる
  - タイトル・本文・タグをまとめて検索対象とする
  - 検索結果はゼロ件の場合に「メモが見つかりません」と表示する

### FR-007: ピン留め

- **説明**: 重要なメモを常にリスト最上部に固定する
- **優先度**: Should
- **受け入れ条件**:
  - メモにピンアイコンが表示され、クリックでピン留め/解除できる
  - ピン留めメモは通常メモより上部にグループ表示される
  - ピン留め中はアイコンが強調表示される

### FR-008: 作成日時・更新日時の表示

- **説明**: メモの作成日時と最終更新日時を表示する
- **優先度**: Should
- **受け入れ条件**:
  - メモ一覧に最終更新日時を表示する（例: `2026/04/20 14:30`）
  - メモ詳細ビューに作成日時・更新日時の両方を表示する

### FR-009: 現在のページURLの自動挿入

- **説明**: メモ作成・編集時にボタン一つで現在閲覧中のページのURLを本文に挿入する
- **優先度**: Should
- **受け入れ条件**:
  - メモ編集エリアに「URLを挿入」ボタンを設置する
  - クリックするとカーソル位置に現在のタブのURLが挿入される

### FR-010: JSONエクスポート / インポート

- **説明**: 全メモデータをJSONファイルとして書き出し・読み込みできる
- **優先度**: Should
- **受け入れ条件**:
  - 設定メニューから「エクスポート」を実行するとJSONファイルがダウンロードされる
  - 「インポート」でJSONファイルを選択すると既存メモにマージされる
  - インポート時に重複するメモID は上書き確認ダイアログを表示する

### FR-011: Markdownプレビュー

- **説明**: メモ本文のMarkdown記法をHTMLとしてプレビュー表示する
- **優先度**: Could
- **受け入れ条件**:
  - 編集モードとプレビューモードをトグルで切り替えられる
  - 対応記法: 見出し / 太字・斜体 / リスト / コードブロック / リンク / 水平線
  - プレビューモードでも編集ボタンで即座に編集に戻れる

### FR-012: Google Keepエクスポート ※Phase 2以降

- **説明**: 選択したメモをGoogle Keepに送信する
- **優先度**: 対象外（Phase 2で検討）
- **備考**: OAuth 2.0外部アプリ・テストモードで実装。社内利用者100名以下を前提とする。

### FR-013: 導入・運用マニュアル

- **説明**: 社内メンバーが自力でインストール・操作できるマニュアルを提供する
- **優先度**: Must
- **受け入れ条件**:
  - デベロッパーモードでのインストール手順をスクリーンショット付きで記載
  - 基本操作（メモの作成・編集・削除・検索）を説明
  - 配布方法（ZIPファイルの共有）を記載
  - バージョンアップ時の更新手順を記載

## 5. 非機能要件


| カテゴリ    | 要件                                   | 計測指標          |
| ------- | ------------------------------------ | ------------- |
| パフォーマンス | サイドバー表示 < 300ms                      | 体感            |
| ストレージ   | メモデータ < 5MB（chrome.storage.local上限内） | 件数目安: 1000件程度 |
| 互換性     | Chrome 最新版 + 1つ前のメジャーバージョン           | 手動確認          |
| セキュリティ  | Manifest V3準拠 / 不要なパーミッション不使用        | コードレビュー       |
| UI      | 日本語UIのみ                              | —             |


---

## 6. 制約・前提条件

- Chrome Manifest V3 を使用する（V2はChromeの廃止スケジュール対象）
- Chrome Web Storeへの公開は行わない（ローカルZIP配布）
- Google Keep連携はPhase 2以降で検討（OAuth 2.0外部アプリ・テストモード、社内100名以下）
- インストールにはChromeのデベロッパーモードを有効にする必要がある

---

## 7. 用語定義


| 用語          | 定義                         |
| ----------- | -------------------------- |
| サイドバー       | ブラウザウィンドウ右端に固定表示されるメモパネル   |
| ピン留め        | メモを一覧の最上部に固定する機能           |
| タグ          | メモに付与する分類用ラベル（`#` プレフィックス） |
| Manifest V3 | Chrome拡張機能の最新仕様バージョン       |
